# Spec: Amazon 广告复盘 Agent

> Spec-Kit 规格说明（做什么 / 为什么）。技术细节见 `plan.md` / `tasks.md`。

## 目标
自动拉取广告数据 → 诊断异常 → 给调价 / 否定词 / 预算建议 → 人类审批后可选执行 → 复盘报告 + 回测。运营的 hero 场景，outcome 最硬（ACOS↓、人效↑），最易量化验证、最易讲成面试故事。

## 用户故事
- **US1 (P0)** — 作为 PPC 专员，我希望每日自动拿到中文复盘报告，指出异常与机会，不用手工拉表。
- **US2 (P0)** — 作为 PPC 专员，我希望 Agent 给出可执行建议（降出价/加否定词/提预算），我一键审批。
- **US3 (P1)** — 作为运营，我希望从搜索词报表自动挖出「有花费无转化」的否定词候选。
- **US4 (P1)** — 作为运营，我希望异常（ACOS 突增、预算耗尽、销量暴跌）实时推送到飞书/企微。
- **US5 (P1)** — 作为品牌主，我希望用历史数据回测：若当初采纳建议，ACOS 会怎样。
- **US6 (P2)** — 作为代运营，我希望多账户/多市场，且品类阈值可定制（3C vs 服饰）。

## 功能需求
- **FR1** MUST 经 `sp-api-mcp-server` 的 `ads_*` 拉多 profile 报表 + 快照。
- **FR2** MUST 计算核心指标：ACOS、ROAS、TACOS、CTR、CVR、CPA、Impression Share、IS lost to budget/rank。
- **FR3** MUST 规则引擎诊断异常（ACOS>目标、花费无单、CTR<基准、预算早耗尽、排名跌、新竞品）。
- **FR4** MUST LLM 对异常生成自然语言归因 + 影响估算。
- **FR5** MUST 生成结构化建议 `{entity, action, old, new, rationale, expected_impact, confidence}`。
- **FR6** MUST 否定词挖掘：搜索词报表筛「花费≥X 且 0 单且无关」→ 候选列表。
- **FR7** MUST 审批网关：默认仅建议；开启自动执行需审批 + 预算/出价护栏。
- **FR8** MUST 复盘报告（NL + Top 5 异常/机会 + 图表）推送飞书/企微。
- **FR9** MUST 回测框架：历史快照模拟建议效果，对比实际。
- **FR10** SHOULD 策略库版本化 + 品类阈值 `category_thresholds.yaml`。

## 验收清单
- [ ] 每日报告生成且含异常诊断。
- [ ] 否定词候选列表准确（无品牌词误伤）。
- [ ] 未开启自动执行时，写操作仅建议不执行。
- [ ] 回测产出吻合率指标。
- [ ] 飞书/企微收到预警与日报。

## 范围外
- DSP；自动执行默认关闭；v1 多账户为 SHOULD。
