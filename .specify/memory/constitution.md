# Constitution — amazon-ads-review-agent

> Spec-Kit 项目治理原则。任何代码前先有 spec → plan → tasks。

## 核心原则（FDE 方法论对齐）
1. **规格驱动**：先 `spec.md` → `plan.md` → `tasks.md`。
2. **可验证 / 可复盘是灵魂**：效果可用历史数据回测，策略沉淀为规则库（高速公路）。对照 FDE 文档 9.4「成果回流平台」。
3. **人类在环 + 预算护栏**：写操作（调价/预算）默认仅建议；自动执行需审批 + 上限（单次预算变更 ≤ 上限、出价 ± ≤ 20%）。
4. **数据隐私优先**：广告报表默认本地；外传需显式开启。
5. **不踩广告政策**：不刷量、不违规定位；否定词不误伤品牌词。
6. **可验证**：回测吻合率、异常召回、ACOS 改善 A/B、人效下降% 可量化。
7. **可复盘**：周度策略复盘、指标追踪、规则库版本化 + 回归测试。
8. **YAGNI / DRY**：v1 仅 SP/SB/SD，不覆盖 DSP；仅建议不自动执行。
9. **TDD**：指标计算、诊断、回测先写失败测试。
10. **工作隔离**：git worktree 并行开发。

## 技术约束
- 语言：Python 3.11+；编排：LangGraph 或简单 DAG；LLM 用于叙事/nuanced 建议；规则引擎做确定性判断。
- 存储：Postgres / DuckDB（快照、建议、效果）；调度：cron / 云函数。
- 默认分支：`main`；提交信息英文祈使句；密钥绝不入库。
